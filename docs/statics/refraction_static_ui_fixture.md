# Refraction Static UI Fixture

This page documents the smoke workflow for the synthetic Static Correction UI
fixture generated by:

```text
scripts/create_refraction_static_ui_fixture.py
```

The fixture exercises this browser flow:

```text
SGY import
  -> direct first-break pick NPZ selection
  -> Static Correction tab execution
  -> job polling
  -> Refraction QC auto-load
```

The browser UI does not create this synthetic data. Generate the files outside
the UI, then load the SGY and pick NPZ through the normal browser controls. The
fixture is not a production-quality geophysical simulation and is not a
replacement for validation with field data.

## Generate The Fixture

Run from the repository root:

```bash
python scripts/create_refraction_static_ui_fixture.py --output-dir /tmp/refraction_static_ui_fixture
```

The script writes:

```text
synthetic_static_2d_one_layer.sgy
predicted_picks_time_s.npz
fixture_metadata.json
expected_static_summary.json
README.md
```

If `segyio` is not installed, the script cannot write
`synthetic_static_2d_one_layer.sgy`. Install `segyio` or run in the repository's
SEG-Y-enabled environment before generating the fixture.

## Smoke Workflow

1. Generate the fixture:

   ```bash
   python scripts/create_refraction_static_ui_fixture.py --output-dir /tmp/refraction_static_ui_fixture
   ```

2. Open `synthetic_static_2d_one_layer.sgy` in the viewer Loader. Use
   `key1=9`, `key2=13`.
3. Open the `Static Correction` tab. The `Target` section should show the
   current viewer file and sort keys.
4. Select `predicted_picks_time_s.npz` directly in `First-break pick NPZ`.
5. Configure geometry, model, output, and export settings.
6. Run Static Correction.
7. Confirm `Refraction QC` auto-loads when the job reaches a ready state.

Users do not need a `file_id` input, pick `job_id`, `artifact_name`, or manual
batch job registration for the primary UI workflow. The Static Correction tab
uses the active viewer target and sends the selected pick NPZ as multipart
field `pick_npz`.

## Static Correction Settings

Open `/`, load the SGY in the viewer, then select `Static Correction`.

Recommended values:

| UI field | Value |
|---|---|
| Target | Current viewer file loaded with `key1=9`, `key2=13` |
| First-break pick NPZ | `/tmp/refraction_static_ui_fixture/predicted_picks_time_s.npz` |
| Linkage | Unchecked / `none` |
| Model preset | `one_layer_global` / `One-layer global V2/T1` |
| V1 | `800 m/s` |
| V2 initial | `2400 m/s` |
| Exports | `canonical_static_table` and `lsst_plus` |
| Register corrected file | Enabled |

Use the fixture geometry header byte mapping:

| Geometry field | Byte |
|---|---:|
| `source_id_byte` | 9 |
| `receiver_id_byte` | 13 |
| `source_x_byte` | 73 |
| `source_y_byte` | 77 |
| `receiver_x_byte` | 81 |
| `receiver_y_byte` | 85 |
| `source_elevation_byte` | 45 |
| `receiver_elevation_byte` | 41 |
| `offset_byte` | 37 |
| `coordinate_scalar_byte` | 71 |
| `elevation_scalar_byte` | 69 |

For the generated fixture, the Static Correction UI `SEG-Y default` geometry
preset should match this fixture geometry. Users should not need to switch to
`custom` unless they intentionally edit the fixture headers.

Use these TraceStore sort keys when loading the SGY in the viewer:

| Sort setting | Byte | Header |
|---|---:|---|
| `key1_byte` | 9 | `source_id_byte` |
| `key2_byte` | 13 | `receiver_id_byte` |

The fixture metadata also records:

```text
recommended_static_correction.key1_byte = 9
recommended_static_correction.key2_byte = 13
recommended_static_correction.min_offset_m = 300.0
recommended_static_correction.max_offset_m = 1800.0
recommended_static_correction.field_corrections.mode = none
```

## Expected Result

After selecting `Run`, the Static Correction tab should show job progress and
then a ready or done job state. The artifact list should appear, and the browser
should auto-load the completed job in the `Refraction QC` tab.

In Refraction QC, expect:

- first-break residuals to be small for the clean synthetic fixture;
- static component plots to be non-zero and smooth;
- a 2D line profile to be available when the run is configured for a supported
  2D projected/profile mode;
- `canonical_static_table` and `lsst_plus` exports to be downloadable.

Use `expected_static_summary.json` as a quick reference for the generated truth
ranges, such as V1, V2, pick-time range, T1 range, weathering thickness, and
weathering correction range.

## Troubleshooting

`segyio` is not installed:
Install `segyio` or run in the SEG-Y-enabled environment before generating the
fixture.

No active viewer file:
Open `synthetic_static_2d_one_layer.sgy` in the viewer Loader before opening or
running the Static Correction tab. The Target section must show the current
viewer file and sort keys.

No pick NPZ selected:
Select `/tmp/refraction_static_ui_fixture/predicted_picks_time_s.npz` in the
`First-break pick NPZ` control. The viewer first-break probability cache is not
a statics pick NPZ.

Sorted order mismatch:
Load the fixture SGY with `key1=9` and `key2=13`. The generated
`predicted_picks_time_s.npz` is written in the same sorted trace order.

Geometry byte mismatch:
Re-enter the byte mapping from `fixture_metadata.json`. Incorrect source,
receiver, coordinate, scalar, elevation, or offset bytes can produce invalid
geometry or poor residuals.

Linkage checkbox accidentally enabled:
Leave linkage unchecked for this fixture. Enabling linkage creates an extra
geometry linkage job and changes the request shape.

V1/V2 velocity order invalid:
V2 must be greater than V1. The fixture recommendation is V1 `800 m/s` and V2
initial `2400 m/s`.

First-break picks outside trace time range:
Regenerate the fixture with the default `--dt-s` and `--n-samples`, or inspect
`fixture_metadata.json` and `expected_static_summary.json` to confirm the pick
times fit inside the trace record.

## Non-Goals

This workflow does not validate real field data, explain GRM or plus-minus
methods, create synthetic data from the browser, or recommend artifact
registration for normal UI runs.
